# GAIA Data Downloader — Agent Instructions

This file provides vendor-neutral coding agent instructions for generating hydroclimatological data download scripts for the GAIA (Geophysical AI-driven Integration and Assimilation) project.

## Purpose

Generate Python download scripts for 10+ environmental data sources used in the GAIA HazLab project. Scripts follow a CONFIG-at-top pattern that separates user-editable parameters from download logic.

## Interaction Pattern

Follow a three-phase approach when generating download scripts:

1. **Parse & Propose** — Identify the data source, extract parameters from the user's request, and propose a structured configuration block for review. Never silently assume required parameters — ask when omitted.
2. **Confirm & Adjust** — Present the configuration with data volume estimates and dependency warnings. Accept user modifications.
3. **Generate & Write** — Generate a Python script with all parameters in a `CONFIG` dict at the top.

## Generated Script Structure

All generated scripts must follow this pattern:

```python
# Generated by GAIA Data Downloader
# Source: [SOURCE]
# Date Range: [start] to [end]

[imports]

# ============================================================
# Configuration — modify these parameters before running
# ============================================================
CONFIG = {
    "source": "[source]",
    "date_range": ("[start]", "[end]"),
    # ... all parameters with comments
}
# ============================================================
# Download logic — generally no need to modify below this line
# ============================================================

def main():
    # 1. Download data (parallel when appropriate)
    # 2. Combine datasets
    # 3. Spatial subset to AOI (if GeoJSON provided)
    # 4. Derive variables (if requested)
    # 5. Save to output format
    # 6. Print QC summary

if __name__ == "__main__":
    main()
```

## Key Warnings

- **HRRR:** Requires `wgrib2` system binary (not pip-installable). Install via conda-forge.
- **ORNL:** Requires NASA Earthdata credentials in `~/.netrc`.
- **Synoptic:** Requires `SYNOPTIC_API_TOKEN` environment variable.
- **WRF-CMIP6:** Uses non-standard time format `YYYY-MM-DD_HH:MM:SS`.
- **USGS:** Returns data in local time zones — convert to UTC.
- **CRS:** Model grids use Lambert Conformal Conic — reproject AOI (Area of Interest) before clipping.

## Standard Pipeline

Every download pipeline follows: configure parameters → download data (often parallel) → spatially subset to AOI → derive variables → save to Zarr → print QC summary. This mirrors the pattern used in the 18 notebooks in the gaia-data-downloaders (https://github.com/gaia-hazlab/gaia-data-downloaders) repository.
